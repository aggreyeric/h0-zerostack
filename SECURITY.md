# Security Policy

## Supported Versions

We provide security updates for the latest version of this project.

| Version  | Supported          |
| -------- | ------------------ |
| latest   | :white_check_mark: |
| < latest | :x:                |

## Reporting a Vulnerability

We take security issues seriously. If you discover a security vulnerability,
**please do not open a public GitHub issue**.

Instead, report it privately through one of the following channels:

- **GitHub Security Advisories:** Use the "Report a vulnerability" button under
  the **Security** tab of this repository.
- **Email:** Send details to **eric@aggreyeric.com**

Please include:
- A description of the vulnerability and its potential impact
- Steps to reproduce the issue
- Any suggested fixes or mitigations

We will acknowledge your report within 72 hours and aim to provide a
substantive response within 7 days.

## Bounty Program

There is **no bug bounty program** for this project. These reports are handled
on a best-effort basis.

## Disclaimer

> ⚠️ This is a **hackathon demo project**, not production software. It has not
> undergone formal security audits. ZeroDeploy fetches and analyzes code from
> public GitHub repositories and forwards it to a third-party LLM provider —
> do not submit private, proprietary, or sensitive repositories. It should not
> be used in production environments without independent review.
